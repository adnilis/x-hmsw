# 项目概述

x-hmsw 是一个纯 Go 实现的高性能向量数据库，专为大规模向量相似度搜索而设计。

## 项目简介

x-hmsw 提供了完整的向量数据库解决方案，包括：
- 多种高效的索引算法（HNSW、IVF、Flat、ANN）
- 灵活的存储引擎选择（Badger、BBolt、Pebble、Memory、Mmap）
- 先进的向量压缩技术（PQ、SQ、Binary）
- 多种文本向量化方法（TF-IDF、BM25、OpenAI Embeddings）
- 完善的监控和日志系统
- 自动保存和备份功能

## 核心特性

### 1. 多索引支持

x-hmsw 支持多种索引算法，可根据场景选择最适合的方案：

- **HNSW (Hierarchical Navigable Small World)**: 基于分层小世界图的近似最近邻搜索，提供高精度和高性能的平衡
- **IVF (Inverted File)**: 基于聚类的倒排索引，适合大规模数据集
- **Flat**: 暴力搜索，提供精确结果，适合小规模数据集
- **ANN**: 近似最近邻搜索，支持 Annoy 和 NGT 实现

### 2. 灵活存储

支持多种存储引擎，满足不同场景需求：

- **BadgerDB**: 基于 LSM 树的高性能键值存储
- **BBolt**: 嵌入式键值存储，适合单机应用
- **PebbleDB**: CockroachDB 的存储引擎，提供优秀的写入性能
- **Memory**: 纯内存存储，提供最快的访问速度
- **Mmap**: 内存映射文件，平衡性能和持久化

### 3. 文本向量化

内置多种文本向量化方法，无需外部依赖：

- **TF-IDF**: 经典的词频-逆文档频率向量化
- **BM25**: 改进的 TF-IDF，包含多种变体（BM25、BM25L、BM25+、BM25F）
- **OpenAI Embeddings**: 支持调用 OpenAI API 获取高质量向量
- **增量更新**: BM25 支持增量更新，无需重新训练

### 4. 向量压缩

通过先进的压缩技术降低内存和存储占用：

- **Product Quantization (PQ)**: 乘积量化，将向量分解为子空间分别量化
- **Scalar Quantization (SQ)**: 标量量化，对每个维度独立量化
- **Binary Quantization**: 二进制量化，将向量转换为二进制表示

### 5. 高性能优化

多种优化手段提升性能：

- **SIMD 加速**: 使用 AVX2、SSE、NEON 指令集加速距离计算
- **对象池**: 复用临时对象，减少内存分配和 GC 压力
- **并发控制**: 细粒度锁和无锁数据结构
- **预取优化**: 提前加载数据，减少缓存未命中

### 6. 易用性

提供简化的 API 接口：

- **QuickDB**: 开箱即用的简化接口，自动保存功能
- **统一接口**: VectorDB 和 Index 接口，易于扩展和替换实现
- **配置灵活**: 支持丰富的配置选项
- **自动保存**: 支持定时自动保存数据

### 7. 监控和日志

完善的监控和日志系统：

- **Prometheus 集成**: 内置监控指标，支持 Prometheus 采集
- **结构化日志**: 使用 uber/zap 提供高性能日志
- **性能指标**: 插入、搜索、删除等操作的延迟和计数

## 技术架构

### 分层设计

```
┌─────────────────────────────────────┐
│         API Layer (QuickDB)         │
├─────────────────────────────────────┤
│      Interface Layer (VectorDB)     │
├─────────────────────────────────────┤
│   Embedding Layer (TF-IDF/BM25)    │
├─────────────────────────────────────┤
│   Index Layer (HNSW/IVF/Flat/ANN)  │
├─────────────────────────────────────┤
│  Compression Layer (PQ/SQ/Binary)   │
├─────────────────────────────────────┤
│   Storage Layer (Badger/BBolt...)   │
├─────────────────────────────────────┤
│     Utils Layer (SIMD/Pool/...)     │
└─────────────────────────────────────┘
```

### 核心组件

1. **API 层**: 提供简化的 QuickDB 接口
2. **接口层**: 定义 VectorDB 和 Index 统一接口
3. **索引层**: 实现各种索引算法
4. **压缩层**: 提供向量压缩功能
5. **存储层**: 实现各种存储引擎
6. **工具层**: 提供通用工具函数

## 快速开始

```go
package main

import (
    "fmt"
    "github.com/adnilis/x-hmsw/api"
    "github.com/adnilis/x-hmsw/types"
)

func main() {
    // 创建向量数据库
    db, err := api.NewQuick("./data")
    if err != nil {
        panic(err)
    }
    defer db.Close()

    // 插入向量
    vectors := []types.Vector{
        {ID: "1", Vector: []float32{0.1, 0.2, 0.3}},
        {ID: "2", Vector: []float32{0.4, 0.5, 0.6}},
    }
    err = db.Insert(vectors)
    if err != nil {
        panic(err)
    }

    // 搜索向量
    query := types.Vector{Vector: []float32{0.1, 0.2, 0.3}}
    results, err := db.Search(query, 5)
    if err != nil {
        panic(err)
    }

    fmt.Println("搜索结果:", results)
}
```
