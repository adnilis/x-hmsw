# 索引算法

x-hmsw 支持多种索引算法，可根据场景选择最适合的方案。

## 索引类型概览

| 索引类型 | 精度 | 性能 | 内存占用 | 适用场景 |
|---------|------|------|---------|---------|
| **HNSW** | 高 | 高 | 中 | 通用场景，平衡精度和性能 |
| **IVF** | 中 | 高 | 低 | 大规模数据集（百万级以上） |
| **Flat** | 最高 | 低 | 低 | 小规模数据集（万级以下） |
| **ANN** | 中 | 高 | 中 | 近似最近邻搜索 |

## 索引算法详解

### HNSW 索引

**HNSW (Hierarchical Navigable Small World)** 是基于分层小世界图的近似最近邻搜索算法。

**特点**:
- 高精度和高性能的平衡
- 支持动态插入和删除
- 适合大多数场景

**核心参数**:
- `M`: 每层最大连接数（默认 16）
- `EfConstruction`: 构建时的搜索宽度（默认 200）
- `EfSearch`: 搜索时的搜索宽度（默认 100）

**适用场景**:
- 通用向量搜索
- 需要高精度和高性能平衡
- 数据规模在百万级以下

**详细文档**: [HNSW索引.md](HNSW索引.md)

### IVF 索引

**IVF (Inverted File)** 是基于聚类的倒排索引算法。

**特点**:
- 内存占用低
- 适合大规模数据集
- 需要预先训练聚类

**核心参数**:
- `NumClusters`: 聚类数量（默认 100）
- `Nprobe`: 搜索时检查的聚类数（默认 10）

**适用场景**:
- 大规模数据集（百万级以上）
- 内存受限环境
- 可以接受一定的精度损失

**详细文档**: [IVF索引.md](IVF索引.md)

### Flat 索引

**Flat** 是暴力搜索算法，计算查询向量与所有向量的距离。

**特点**:
- 精度最高（精确搜索）
- 性能随数据量线性下降
- 无需训练

**适用场景**:
- 小规模数据集（万级以下）
- 需要精确结果
- 数据更新频繁

**详细文档**: [Flat索引.md](Flat索引.md)

### ANN 索引

**ANN (Approximate Nearest Neighbor)** 是近似最近邻搜索算法的统称。

**特点**:
- 高性能
- 可配置的精度-性能权衡
- 支持多种底层实现（Annoy、NGT）

**实现**:
- **Annoy**: Spotify 开发的近似最近邻库
- **NGT**: 高维数据的近似最近邻搜索

**适用场景**:
- 需要高性能搜索
- 可以接受一定的精度损失
- 实时应用

**详细文档**: [ANN索引.md](ANN索引.md)

## 如何选择索引类型

### 根据数据规模

| 数据规模 | 推荐索引 |
|---------|---------|
| < 10,000 | Flat |
| 10,000 - 1,000,000 | HNSW |
| > 1,000,000 | IVF 或 HNSW |

### 根据精度要求

| 精度要求 | 推荐索引 |
|---------|---------|
| 精确搜索 | Flat |
| 高精度 | HNSW |
| 中等精度 | IVF |
| 可接受精度损失 | ANN |

### 根据性能要求

| 性能要求 | 推荐索引 |
|---------|---------|
| 最高性能 | IVF |
| 高性能 | HNSW |
| 中等性能 | Flat |

### 根据内存限制

| 内存限制 | 推荐索引 |
|---------|---------|
| 内存充足 | HNSW |
| 内存受限 | IVF |
| 内存非常受限 | Flat |

## 性能对比

基于以下配置的性能测试：
- 向量维度: 128

## 使用示例

### 使用 HNSW 索引

```go
config := iface.Config{
    Dimension:      128,
    IndexType:      iface.HNSW,
    M:              16,
    EfConstruction: 200,
}

db, _ := iface.NewPureGoVectorDB(config)
```

### 使用 IVF 索引

```go
config := iface.Config{
    Dimension:    128,
    IndexType:    iface.IVF,
    NumClusters:  100,
    Nprobe:       10,
}

db, _ := iface.NewPureGoVectorDB(config)
```

### 使用 Flat 索引

```go
config := iface.Config{
    Dimension: 128,
    IndexType: iface.Flat,
}

db, _ := iface.NewPureGoVectorDB(config)
```

## 索引参数调优

### HNSW 参数调优

**M (每层最大连接数)**:
- 增大 M: 提高精度，降低性能，增加内存
- 减小 M: 降低精度，提高性能，减少内存
- 推荐值: 16-32

**EfConstruction (构建时的搜索宽度)**:
- 增大 EfConstruction: 提高精度，降低构建速度
- 减小 EfConstruction: 降低精度，提高构建速度
- 推荐值: 100-400

**EfSearch (搜索时的搜索宽度)**:
- 增大 EfSearch: 提高精度，降低搜索速度
- 减小 EfSearch: 降低精度，提高搜索速度
- 推荐值: 50-200

### IVF 参数调优

**NumClusters (聚类数量)**:
- 增大 NumClusters: 提高精度，增加内存和训练时间
- 减小 NumClusters: 降低精度，减少内存和训练时间
- 推荐值: sqrt(N)，其中 N 是向量数量

**Nprobe (搜索时检查的聚类数)**:
- 增大 Nprobe: 提高精度，降低搜索速度
- 减小 Nprobe: 降低精度，提高搜索速度
- 推荐值: NumClusters / 10

## 最佳实践

1. **小数据集**: 使用 Flat 索引，获得精确结果
2. **中等数据集**: 使用 HNSW 索引，平衡精度和性能
3. **大数据集**: 使用 IVF 索引，降低内存占用
4. **实时应用**: 使用 HNSW 或 IVF，获得高性能
5. **离线分析**: 使用 Flat 索引，获得精确结果
