# 向量压缩

x-hmsw 支持多种向量压缩技术，可显著降低内存和存储占用。

## 压缩算法概览

| 压缩算法 | 压缩比 | 精度损失 | 计算开销 | 适用场景 |
|---------|-------|---------|---------|---------|
| **PQ** | 高 | 中 | 中 | 大规模数据集 |
| **SQ** | 中 | 低 | 低 | 通用场景 |
| **Binary** | 最高 | 高 | 低 | 极限压缩 |

## 压缩算法详解

### 乘积量化 (PQ)

**Product Quantization (PQ)** 将向量分解为多个子空间，对每个子空间分别进行量化。

**特点**:
- 高压缩比
- 中等精度损失
- 需要训练阶段
- 适合大规模数据集

**核心参数**:
- `m`: 子量化器数量
- `k`: 每个子量化器的聚类数

**适用场景**:
- 大规模数据集
- 内存受限环境
- 可以接受中等精度损失

**详细文档**: [乘积量化.md](乘积量化.md)

### 标量量化 (SQ)

**Scalar Quantization (SQ)** 对向量的每个维度独立进行量化。

**特点**:
- 中等压缩比
- 低精度损失
- 无需训练
- 适合通用场景

**核心参数**:
- `bits`: 每个维度的位数（8、16、32）

**适用场景**:
- 通用场景
- 需要快速部署
- 精度要求较高

**详细文档**: [标量量化.md](标量量化.md)

### 二进制量化

**Binary Quantization** 将向量转换为二进制表示。

**特点**:
- 最高压缩比
- 高精度损失
- 计算简单
- 适合极限压缩

**适用场景**:
- 极限压缩需求
- 可以接受高精度损失
- 快速近似搜索

**详细文档**: [二进制量化.md](二进制量化.md)

## 如何选择压缩算法

### 根据压缩比需求

| 压缩比需求 | 推荐算法 |
|-----------|---------|
| 最高压缩比 | Binary |
| 高压缩比 | PQ |
| 中等压缩比 | SQ |

### 根据精度要求

| 精度要求 | 推荐算法 |
|---------|---------|
| 高精度 | SQ |
| 中等精度 | PQ |
| 可接受精度损失 | Binary |

### 根据部署复杂度

| 部署复杂度 | 推荐算法 |
|-----------|---------|
| 简单部署 | SQ, Binary |
| 需要训练 | PQ |

## 使用示例

### 使用标量量化

```go
// 创建标量量化器
sq := sq.NewScalarQuantizer(128, 8)  // 128 维，8 位

// 训练量化器
vectors := [][]float32{...}
sq.Train(vectors)

// 编码向量
encoded := sq.Encode(vector)

// 解码向量
decoded := sq.Decode(encoded)
```

### 使用乘积量化

```go
// 创建乘积量化器
pq := pq.NewProductQuantizer(128, 8, 256)  // 128 维，8 个子空间，256 个聚类

// 训练量化器
vectors := [][]float32{...}
pq.Train(vectors)

// 编码向量
encoded := pq.Encode(vector)

// 解码向量
decoded := pq.Decode(encoded)
```

### 使用二进制量化

```go
// 创建二进制量化器
bq := binary.NewBinaryQuantizer(128)

// 编码向量
encoded := bq.Encode(vector)

// 解码向量
decoded := bq.Decode(encoded)
```

## 压缩算法参数调优

### 标量量化参数调优

**bits (每个维度的位数)**:
- 8-bit: 压缩比 4x，精度损失小
- 16-bit: 压缩比 2x，精度损失极小
- 32-bit: 无压缩

推荐值: 8-bit

### 乘积量化参数调优

**m (子量化器数量)**:
- 增大 m: 提高精度，增加计算开销
- 减小 m: 降低精度，减少计算开销
- 推荐值: 8-16

**k (每个子量化器的聚类数)**:
- 增大 k: 提高精度，增加内存和训练时间
- 减小 k: 降低精度，减少内存和训练时间
- 推荐值: 256

### 二进制量化参数调优

二进制量化通常不需要参数调优。

## 压缩与索引的结合

压缩算法可以与索引算法结合使用：

```go
// 使用 HNSW 索引 + PQ 压缩
config := iface.Config{
    Dimension:      128,
    IndexType:      iface.HNSW,
    Compression:    iface.PQ,
    CompressionOptions: map[string]interface{}{
        "m": 8,
        "k": 256,
    },
}

db, _ := iface.NewPureGoVectorDB(config)
```

## 最佳实践

1. **通用场景**: 使用 SQ (8-bit)，平衡压缩比和精度
2. **大规模数据集**: 使用 PQ，高压缩比
3. **极限压缩**: 使用 Binary，最高压缩比
4. **高精度要求**: 不使用压缩或使用 SQ (16-bit)
5. **快速部署**: 使用 SQ 或 Binary，无需训练
